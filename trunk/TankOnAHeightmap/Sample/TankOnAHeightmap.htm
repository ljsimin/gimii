TankOnAHeightmap

This sample demonstrates how to use information about a heightmap's vertex normals to follow the contour of the terrain.

Sample Overview

The Heightmap Collision sample demonstrated how to make a sphere roll around on a heightmap. One way to improve this is by orienting the object to follow the contour of the terrain. This sample provides a more believable effect for objects such as vehicles, which will not always be pointing the same way up as they move around.

This sample demonstrates how to do this by moving a tank around on a heightmap. The code involved is similar to the Heightmap Collision sample, and throughout this sample it is assumed that you are familiar with the Heightmap Collision sample. In addition, you will use the technique described in the Simple Animation sample to animate the tank.

How the Sample Works

As previously mentioned, the sample builds on the Heightmap Collision sample. To support the new functionality, the code has been changed primarily in two areas. First, the HeightMapInfoContent class had to be changed to get not only heights, but also normals when building.

This code is relatively straightforward. HeightMapInfoContent's constructor now takes in a MeshContent object. It creates two 2D arrays, one of heights and one of normals. It loops through every vertex in the mesh, and examines its position and normal. From the position's X and Z values, it can tell where in the arrays to place the vertex's normal and height.

The second major code change is in Tank.cs, and is the code that orients the tank to the terrain. To calculate a final orientation matrix from the terrain's normal, the code employs a series of cross products. Cross products are useful in 3D math, and this sample assumes that you are familiar with how they work. If not, wikipedia is a good resource. In short, a cross product is a function that takes two vectors, and returns a vector that is perpendicular to both of them.

So, how do you calculate the tank's orientation for a point on the heightmap? In order to make an orientation matrix, you need three unit vectors, one in each XYZ direction. You want your orientation's Up vector to be the terrain's normal, so that covers one axis.

The other two can be calculated using cross products. The diagram below shows the direction the user wants to be facing (Forward) and the terrain's normal (Normal). Since the cross product produces a vector that is perpendicular to both vectors, you can calculate the Right vector.

The final step is to find the Forward direction of your orientation matrix. It's tempting to just take the user's desired forward vector, but this won't work. If you look at the diagram above, it's easy to see why: that vector is always in the XZ plane. If you used that value as the Forward for your matrix, you wouldn't have a valid orientation matrix. The basis vectors of orientation matrices must be orthonormal, which is a fancy way of saying they have to be at right angles to one another, and one unit long.

So, how can you calculate the Forward vector? It's actually a lot simpler than it sounds. To find forward, you just calculate the cross product of Up and Right. By normalizing the vectors as you calculate them, you ensure that our finished orientation matrix is orthonormal, and can be used to correctly draw the tank on the terrain.

Extending the Sample

It's important to realize that this method for orienting an object on the terrain isn't perfect. Our approach pretends that the tank makes contact with the ground at only one point in its center. Obviously, this is not true, and this simplification can become visible on steep patches of terrain. In these cases, the wheels' height will not match that of the terrain, and may float above or clip below the terrain. You could better simulate the tank by giving it four contact points; one at the center of each of the wheels. However, calculating the orientation of the tank using this data is a much more complicated problem. Our approach is simpler, both in terms of code and calculations performed, and the illusion is broken in only a few cases.

With a few other additions, these problems can be masked and the illusion can be made more convincing. One thing that would help a lot would be shadows. Shadows add a lot of depth to a scene, and are very good at "planting" an object on the ground. This can reduce the feeling that the tank is floating along the top of the ground.

The second thing that would help would be particles. Remember that our algorithm that calculates heights and normals is not triangle perfect, and in some cases the tank can dip under the terrain. We have masked this by keeping the heightmap as tesselated as possible. As the triangles grow larger and larger, the problems become more apparent. Adding dust particles around the wheels would mask the problem even more. This, like the shadows, would also help to plant the tank on the ground.
